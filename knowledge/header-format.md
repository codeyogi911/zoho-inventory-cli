# Authorization header is `Zoho-oauthtoken`, not `Bearer`

Zoho's API uses a custom keyword in the Authorization header:

```
Authorization: Zoho-oauthtoken <access_token>
```

This is documented at `https://www.zoho.com/inventory/api/v1/oauth/`. The standard `Bearer <token>` keyword is rejected by Zoho's gateway with HTTP 401.

In `.clify.json` the auth scheme is recorded as `bearer` for validation-gate compatibility — but the wire prefix in `lib/auth.mjs` is `Zoho-oauthtoken`. Don't "fix" the auth header to use `Bearer` — it'll break every request.

If you see `auth_invalid` (HTTP 401) on a token you know is valid, the most common cause is the `Bearer` prefix sneaking in (e.g., from copying curl examples generated by tools that assume Bearer auth).
